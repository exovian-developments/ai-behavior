# Command: `/ai-behavior:foundation-create [product-name]`

**Status:** ‚úÖ DESIGNED

---

## Overview

**Purpose:** Generate a product foundation document by compacting feasibility analysis outputs into validated facts, enriched problem statements, re-classified capabilities, and financial benchmarks. Bridges the gap between raw feasibility research and product blueprint design.

**Schema:** `ai_files/schemas/product_foundation_schema.json`

**Input:** `ai_files/*_feasibility.json` (required)

**Output:** `ai_files/product_foundation.json`

**Parameters:** `[product-name]` (optional ‚Äî if not provided, inferred from feasibility)

**Key Features:** Feasibility compaction, problem enrichment with evidence, capability re-classification (revenue-impact ‚Üí operational criticality), Monte Carlo & Bayesian summary extraction, SWOT synthesis, vision & mission drafting, blueprint readiness gate

---

## Flow Derivations

| Flow | Condition | Description |
|------|-----------|-------------|
| **Flow A** | Feasibility file found with complete data | Full compaction: extract, enrich, synthesize, validate readiness |
| **Flow B** | Feasibility file found with partial data | Partial compaction: extract what exists, mark gaps, assess readiness with warnings |

     - No posteriors or very low ‚Üí "anecdotal"

     - Moderate posteriors with market data ‚Üí "measured"
     - High posteriors or pre-sales ‚Üí "validated"
