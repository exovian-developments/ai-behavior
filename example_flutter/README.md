# Flutter Calculator App Example

This directory demonstrates how **ai-behavior** schemas work in a Flutter mobile application.

## About This Example

**Project:** Simple Calculator App
**Stack:** Flutter 3.29 + Dart 3.3 + Riverpod
**Architecture:** Clean Architecture (Presentation → Domain → Data)

### What's Demonstrated

1. **Project Manifest** (`ai_files/project_manifest.json`)
   - Complete technical stack documentation
   - Architecture patterns across 3 layers
   - Feature list for calculator operations

2. **Project Rules** (`ai_files/project_rules.json`)
   - Layer-specific coding standards
   - Flutter best practices
   - Naming conventions

3. **Development Logbook** (`ai_files/logbooks/CALC-001.json`)
   - Real workflow for implementing calculator
   - Context tracking across development
   - Demonstrates AI agent collaboration

## Structure

```
example_flutter/
├── ai_files/
│   ├── schemas/              # JSON schemas
│   ├── logbooks/
│   │   └── CALC-001.json     # Calculator implementation logbook
│   ├── project_manifest.json # Project context
│   ├── project_rules.json    # Coding standards
│   └── user_pref.json        # User preferences
├── lib/src/                  # App structure reference
├── pubspec.yaml              # Dependencies
└── README.md                 # This file
```
