# Migration v2.5.0 → v3.0.0

## Narrative

```
WAVES 3.0 — Plugin-first distribution

The framework (commands, hooks, agents, skills, schemas) now lives in the
Claude Code plugin and updates with /plugin update. Your project no longer
carries framework copies — only its own artifacts in ai_files/. Upgrades are
a single /plugin update for all your projects at once, and your
~/.claude/settings.json is never touched by Waves.
```

## Steps

1. **Version marker relocation.** If `.claude/waves-version` exists (the legacy 2.x
   location), read its value, then delete that file. The applied version now lives in
   `ai_files/.waves-version` (a git-committable artifact) — `/waves:upgrade` writes it in
   its final step, so you only need to remove the legacy file here. If `.claude/waves-version`
   does not exist, skip this step.

2. **No artifact schema changes.** Between 2.5.0 and 3.0.0 the blueprint, project_rules,
   roadmap, and logbook schemas are unchanged. No field patches are required on existing
   artifacts.

3. **Legacy framework files are NOT removed here.** Removing `.claude/commands/waves:*.md`,
   `.claude/hooks/waves-*.sh`, and `ai_files/schemas/` is a one-time operation handled by
   `/waves:migrate-from-v2-to-v3`. Do not delete framework files from this migration.
