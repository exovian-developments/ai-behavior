## File Naming Convention

Each subagent file follows the pattern:
```
[number]-[subagent-name].md
```

Example:
```
01-project-initializer.md
02-manifest-creator-new-software.md
...
```

---

## Subagent File Structure

Each subagent file should contain:

```markdown
# Subagent: [name]

## Purpose
[What this subagent does]

## Used By
[List of commands that use this subagent]

## Tools Available
[List of tools: Read, Write, Glob, Grep, Bash, Task, etc.]

## Input
[What the main agent passes to this subagent]

## Output
[What this subagent returns to the main agent]

## Instructions
[Detailed instructions for the subagent]

## Example Interaction
[Example of how the subagent interacts with user/main agent]
```
